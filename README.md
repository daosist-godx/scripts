# Scripts

Набор небольших админских тулз и хелперов под повседневную работу.

Тут лежат скрипты, батники и небольшие наборы для типовых задач: быстро просканить подсеть, отправить сообщение пользователю, зацепиться к сессии через shadow, поднять удаленный поиск файлов и не только.

Репа без лишнего пафоса: рабочие штуки, которые можно быстро открыть, понять и запустить.

## Что есть сейчас

### `network_scan.ps1`
PowerShell-тулза для сканирования подсети `/24`.

Что умеет:
- пингует адреса `1..254`;
- пытается определить имя ПК;
- пытается вытащить текущего пользователя;
- если получается, тянет `DisplayName` из AD;
- сохраняет результат в `TXT` или `CSV`.

Запуск:

```powershell
powershell -ExecutionPolicy Bypass -File .\\network_scan.ps1
```

Подробности: `docs/network_scan.md`

---

### `MSG (запуск от админа).bat`
Небольшая бат-тулза для отправки сообщения в пользовательскую сессию Windows.

Что делает:
- спрашивает IP удаленного ПК;
- показывает сессии через `quser`;
- дает выбрать `Session ID` или `*`;
- отправляет сообщение через `msg.exe`.

Запуск:

```bat
MSG (запуск от админа).bat
```

Подробности: `docs/msg.md`

---

### `RDS shadow (запуск от админа).bat`
Короткий хелпер для shadow-подключения к уже открытой RDP-сессии.

Что делает:
- спрашивает IP удаленного ПК;
- показывает сессии через `query session`;
- просит ID нужной сессии;
- запускает `mstsc /shadow` с управлением.

Запуск:

```bat
RDS shadow (запуск от админа).bat
```

Подробности: `docs/rds-shadow.md`

---

### `Поиск запрещенных файлов у пользователей/`
Не одиночный скрипт, а небольшой набор тулз для удаленного запуска `Everything` и поиска файлов на целевом ПК.

Внутри:
- BAT для запуска;
- `PsExec`;
- portable-сборка `Everything`;
- отдельный README по настройке.

Подробности: `docs/remote-files-search.md`

---

### `psexec/`
Отдельная папка под запуск `cmd.exe` на удаленном хосте через `PsExec`.

Внутри:
- `PSEXEC (запуск от админа).bat` — твоя исходная версия без правок;
- `PsExec.exe` — сама утилита;
- `README.md` — короткая шпаргалка по запуску и ключам.

Подробности: `psexec/README.md`

## Как дальше поддерживать репу

Чтобы тут не началась каша, логика простая:
- сами скрипты оставляем как рабочие артефакты;
- для каждой новой тулзы делаем короткий README;
- если у скрипта есть дополнительные файлы или portable-набор, лучше складывать его в отдельную папку;
- реальные креды, временные базы, логи и рабочие артефакты в репу не тащим.
