# Scripts

Набор небольших Windows- и PowerShell-утилит для админских задач.

Этот файл — предлагаемая версия корневого README, чтобы постепенно привести репозиторий к более аккуратному виду без резкой ломки текущей структуры.

## Что внутри сейчас

- `network_scan.ps1` — сканирование подсети `/24` с попыткой получить имя ПК, логин и `DisplayName`.
- `MSG (запуск от админа).bat` — отправка сообщения в выбранную пользовательскую сессию.
- `RDS shadow (запуск от админа).bat` — подключение к существующей RDP-сессии через shadow.
- `Поиск запрещенных файлов у пользователей/` — набор для удаленного запуска `Everything` через `PsExec`.
- `psexec/` — отдельная папка с оболочкой для запуска удаленной `cmd.exe` через `PsExec`.

## Как я бы вел репозиторий дальше

Для новых скриптов лучше придерживаться одного шаблона:

- если скрипт маленький и автономный — оставлять файл в корне и делать описание в `docs/`;
- если скрипт тянет за собой утилиты, конфиги или дополнительные файлы — выносить в отдельную папку;
- для каждого инструмента иметь короткое описание: что делает, как запускать, особенности, ограничения;
- не хранить в репозитории реальные логины, пароли, историю запусков и рабочие базы;
- артефакты результатов (`*.csv`, `*.txt`, логи, временные файлы) держать в `.gitignore`.

## Рекомендуемая структура на будущее

```text
scripts/
├─ README.md
├─ docs/
│  ├─ network_scan.md
│  ├─ msg.md
│  ├─ rds-shadow.md
│  ├─ remote-files-search.md
│  └─ _template_script_readme.md
├─ network_scan.ps1
├─ MSG (запуск от админа).bat
├─ RDS shadow (запуск от админа).bat
├─ Поиск запрещенных файлов у пользователей/
└─ psexec/
   ├─ PSEXEC (запуск от админа).bat
   └─ README.md
```

## Почему такой подход удобен

- проще быстро понять, что есть в репозитории;
- легче добавлять новые скрипты без каши;
- у каждого инструмента есть короткая инструкция;
- большие наборы не смешиваются с одиночными BAT/PS1-файлами.
